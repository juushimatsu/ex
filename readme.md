# Сервис техподдержки

Установка зависимостей: `npm install`

Запуск:
- Основной сервер: `npm start` (порт 3000)
- Сервис изображений: `npm run start:image` (порт 3001)
- Оба сервиса: `npm run start:all`

Аккаунты агентов:
- login: `agent1`, password: `pass1`
- login: `agent2`, password: `pass2`

## Архитектура

### Схема взаимодействия сервисов

```
┌──────────────┐       ┌──────────────────┐       ┌──────────────────┐
│   Клиент     │──────>│  Основной сервер  │──────>│ Сервис           │
│   (браузер)  │<──────│  (Express, :3000) │<──────│ изображений      │
└──────────────┘       │                   │       │ (Express, :3001) │
                       │  - Сессии         │       │                  │
┌──────────────┐       │  - Тикеты         │       │  - Хранение      │
│   Агент      │──────>│  - Сообщения      │       │  - Отдача        │
│   (браузер)  │<──────│  - Авторизация    │       │  - Валидация     │
└──────────────┘       │  - WebSocket      │       │    подписей      │
                       └──────────────────┘       └──────────────────┘
```

### Протоколы обмена

- **HTTP REST API** — создание сессий, тикетов, авторизация, загрузка изображений
- **WebSocket** — обмен сообщениями в реальном времени, уведомления о новых тикетах

### Механизмы аутентификации и авторизации

- **Клиенты**: получают уникальный анонимный `sessionId` при первом обращении (POST /api/session). Опционально могут указать ФИО/адрес/паспорт.
- **Агенты**: авторизация через логин/пароль (POST /api/auth/login), получают Bearer-токен для всех последующих запросов.
- **WebSocket**: клиент инициализирует подключение с `client_init` + `sessionId`, агент — с `agent_init` + `token`. Без валидных учётных данных отправка сообщений невозможна.

### Защита от флуда и злоупотреблений

- **HTTP Rate Limiter**: ограничение количества запросов на endpoint (настраиваемые лимиты: например, 10 логинов/мин, 30 загрузок/мин)
- **WebSocket Rate Limiter**: ограничение 50 сообщений за 5 секунд на соединение
- **Лимит длины сообщения**: макс. 4000 символов
- **Лимит активных тикетов на агента**: не более 10 параллельных активных обращений
- **Лимит размера файла**: макс. 10 МБ, только JPEG/PNG/GIF/WebP
- **Защита от сообщений в закрытый тикет**: сервер отклоняет попытки отправить сообщение в закрытое обращение

### Обеспечение приватности картинок

Картинки хранятся на **отдельном сервисе** (порт 3001) и недоступны по прямому URL. Доступ реализован через **HMAC-подписанные URL**:

1. При загрузке изображения основной сервер проверяет, что загрузчик — участник тикета
2. Основной сервер пересылает файл на сервис изображений (base64)
3. При запросе картинки основной сервер генерирует подписанный URL:
   - В URL включены: `imageId`, `ticketId`, `expires` (время истечения), `sig` (HMAC-SHA256 подпись)
   - Подпись вычисляется как `HMAC-SHA256(imageId:ticketId:expires, SECRET_KEY)`
4. Сервис изображений при отдаче проверяет:
   - Наличие всех параметров подписи
   - Срок действия URL не истёк
   - HMAC-подпись валидна
   - ticketId в URL совпадает с ticketId, привязанным к картинке
5. Заголовки ответа: `Cache-Control: private, no-store`, `X-Content-Type-Options: nosniff`
6. Картинки автоматически удаляются через 24 часа

Без знания секретного ключа невозможно подделать подпись и получить доступ к чужим изображениям.

### Переменные окружения

- `SIGNING_SECRET` — секретный ключ для HMAC-подписей (по умолчанию: `support-secret-key-change-me`)
- `IMAGE_SERVICE_URL` — URL сервиса изображений (по умолчанию: `http://localhost:3001`)
